Я реализовал API для бронирования мест на мероприятие с использованием PostgreSQL в качестве основной базы данных.

Чтобы упростить запуск и избежать ручной настройки окружения, я добавил Docker, который позволяет поднять всё приложение одной командой

После запуска можно сразу отправлять HTTP-запросы к API, не устанавливая PostgreSQL и Node.js вручную.

Запуск
```
docker-compose up --build
```
После сборки сервер доступен по адресу:
```
http://localhost:3009
```

Доступные API
1. Создание события

POST /api/events/create

Тело запроса:
```
{
  "name": "smth",
  "totalSeats": 100
}
```
Ответ:
```
{
  "id": 1,
  "name": "smth",
  "totalSeats": 100
}
```
2. Бронирование места

POST /api/bookings/reserve

Тело запроса:
```
{
  "eventId": 1,
  "userId": "user123"
}
```
Успешный ответ:
```
{
  "id": 5,
  "userId": "user123",
  "event": { "id": 1 },
  "createdAt": "2025-11-04T12:45:00.000Z"
}
```

Прошу заметить что проверки на уникальность или валидация данных по бизнес-логике в API/EVENT/CREATE НЕ РЕАЛИЗОВАНЫ, так как цель — просто иметь возможность создать событие для дальнейшего бронирования.
